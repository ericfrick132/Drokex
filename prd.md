APÉNDICE A: ALCANCE DEL TRABAJO
1. DESCRIPCIÓN DEL PROYECTO
Nombre del Proyecto: DROKEX
Descripción General: Desarrollo de una plataforma web para conectar empresas proveedoras con compradores potenciales. La plataforma permitirá a las empresas registrarse, subir su inventario con imágenes y gestionar sus productos de manera sencilla. Esta primera fase (MVP) se enfoca en validación de mercado con funcionalidades básicas.
Usuarios Finales:
Empresas proveedoras/exportadoras colombianas
Compradores/importadores potenciales
Administradores de la plataforma
2. OBJETIVOS DEL PROYECTO
Los objetivos principales del MVP son:
Crear una plataforma funcional para registro de empresas proveedoras
Permitir la gestión básica de inventarios con imágenes
Generar un catálogo público visible para compradores potenciales
Validar el modelo de negocio con usuarios reales
Establecer una base escalable para futuras funcionalidades
3. REQUISITOS FUNCIONALES - FASE MVP
3.1 Landing Page de Pre-lanzamiento
Página de inicio atractiva con información del servicio
Formulario de captación de leads para empresas interesadas
Diseño responsive y profesional
3.2 Sistema de Registro de Empresas
Formulario de registro simple y rápido
Campos básicos: nombre empresa, contacto, email, teléfono
Validación de correo electrónico
Proceso de aprobación por administrador
3.3 Gestión de Inventarios
Subir productos con múltiples imágenes
Campos: nombre producto, descripción, precio, categoría
Editar información de productos existentes
Activar/desactivar productos
Vista previa del producto
3.4 Catálogo Público
Vista de todos los productos activos
Búsqueda básica por nombre o categoría
Filtros simples
Vista detallada de cada producto
Información de contacto de la empresa proveedora
3.5 Panel de Administración
Login seguro para administradores
Aprobar/rechazar empresas registradas
Ver estadísticas básicas (empresas, productos)
Gestión básica de contenido
4. REQUISITOS TÉCNICOS
4.1 Plataforma de Desarrollo
Frontend: React.js con diseño responsive
Backend: .NET Core API
Base de datos: PostgreSQL
Hosting: AWS (Digital Ocean para MVP)

4.3 Integraciones MVP
Sistema de emails para notificaciones
Almacenamiento de imágenes en la nube

7. PROCESO DE ACEPTACIÓN
7.1 Criterios de Aceptación
Todas las funcionalidades descritas operativas
Sin errores críticos de funcionamiento
Diseño implementado según lo acordado
Plataforma accesible desde navegadores modernos
7.2 Procedimiento de Pruebas
Pruebas funcionales por parte del desarrollador
Período de prueba de 48 horas para el cliente
Reporte de issues encontrados
Corrección de bugs críticos
7.3 Proceso de Revisión y Aprobación
Demostración en vivo de la plataforma
Revisión de checklist de funcionalidades
Aprobación formal por escrito
8. SOPORTE POST-IMPLEMENTACIÓN
8.1 Período de Garantía
Duración: 30 días calendario desde la entrega
Alcance: Corrección de bugs y errores de funcionamiento
No incluye: Nuevas funcionalidades o cambios de diseño
8.2 Mantenimiento
Mantenimiento mensual disponible por contrato separado
Incluye: actualizaciones de seguridad, backups, monitoreo
Costo: A cotizar según necesidades
9. EXCLUSIONES DEL MVP
Esta fase NO incluye:
Integración con pasarelas de pago
Chat interno entre usuarios
Integraciones con courriers
Sistema de wallets o transacciones
Integración con redes sociales
Funcionalidades de comercio electrónico avanzadas
